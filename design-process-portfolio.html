American Century Investments

Modernizing Financial Tools Through Design Systems & CMS Migration

Product Designer — Design System, Client Portal, CMS Migration, App Features

1 Discover

Understanding the legacy landscape

American Century Investments needed to modernize their digital presence across web and mobile platforms. The existing system relied on a legacy CMS with inconsistent components, making it difficult for content teams and developers to work efficiently. I began by auditing the existing client portal, landing pages, and internal ESG dashboard to map pain points across user journeys.

Key Finding

The lack of a unified design system meant each platform (Android, iOS, desktop web) had divergent UI patterns — creating confusion for users who used multiple devices and slowing down the engineering team significantly.

Beachbody on Demand

Pioneering a Live Workout Experience Across Every Screen

Product Designer — BODcast Experience, Live Video Player, Cross-Platform App Design

1 Discover

Defining the live fitness experience from scratch

Beachbody was launching BODcast — a brand new live workout experience that had never existed before. Users could join real-time classes from live studios, see other participants, interact with trainers, and work out together from home. The challenge was designing an experience that felt as energizing as being in a physical class, across Android, iOS, and browser web simultaneously.

Core Challenge

No existing pattern existed for this type of experience. We were building something from zero — a live multi-participant video workout with studio integration, music playback, real-time casting, and social interaction — all while the user is physically exercising and needs glanceable, minimal-friction UI.

🔍
Replace: Research / competitive landscape analysis
2 Ideate

Exploring live video player concepts & interaction models

I explored dozens of concepts for the live class interface — how to show the trainer prominently while still displaying a grid of fellow participants, how to handle music controls during an intense workout, and how the "casting" feature (switching your view to the studio camera) should work. Every decision had to account for the fact that users are mid-exercise — large tap targets, high contrast, minimal cognitive load.

3 Design

Designing the live class video player & participant grid

The final design featured a bold, immersive video player with the trainer front-and-center, surrounded by a dynamic participant grid showing fellow workout-goers in real time. I designed the music overlay (featuring track info from the studio's playlist), the "Leave Class" flow, and the casting control that let users switch between their camera and the studio view. The entire system was designed in Figma with detailed prototypes demonstrating transitions and states.

4 Iterate

Coordinating with live studios & real-world testing

This wasn't just a digital product — it had to work in tandem with actual live studios. I worked closely with the studio operations team and engineers to iterate on the experience based on real class tests. We refined the participant grid layout, adjusted the trainer video sizing for different screen ratios, and optimized the casting transition so it felt seamless rather than jarring during a high-energy workout.

Cross-platform specs for Android, iOS, and Web

Because this shipped simultaneously on three platforms, handoff documentation was critical. I created platform-specific specs in Figma that accounted for native patterns (iOS navigation bars vs Android material patterns) while maintaining visual consistency. The live video player component specs included detailed state management documentation — connecting, buffering, live, reconnecting, and ended states — all with pixel-perfect specifications.

Ring (Amazon)

Supporting Ring's Headless CMS Migration

Technical Design Support — CMS Migration, Component Specs, Developer Handoff

1 Discover

Helping migrate Ring's web component ecosystem

My role at Ring was focused on technical migration support. The team was moving website components into a new headless CMS, and I helped document existing component patterns, compare legacy styles against target states, and prepare clearer specs for developers.

Problem Statement

The legacy CMS had accumulated years of one-off component patterns. The migration needed clear documentation so developers could tell what to preserve, what to update, and how each component should behave in responsive layouts.

🔍
Replace: Component audit / legacy system analysis
2 Ideate

Clarifying reusable component needs

I supported the team in sorting reusable web patterns such as navigation, hero areas, product modules, CTAs, and footers. The goal was to make the migration easier for engineering by separating global component behavior from page-specific content.

📌
Replace: Component architecture / planning diagram
3 Design

Translating visual patterns into developer specs

I created and updated Sketch specs for component properties like sizing, typography, spacing, styling, and responsive behavior. The work was intentionally technical: make the existing web experience easier to migrate without asking engineers to infer details from screenshots alone.

🎨
Replace: Design system specs with sizing/typography/spacing (Recommended: 1600x900px)
Headless CMS Approach

The most useful design contribution was not a new visual concept. It was making component rules explicit enough for a headless CMS environment, where content could change but structure, spacing, and behavior still needed to stay consistent.

4 Iterate

Checking component flexibility across page types

I helped review how migrated components would hold up across product pages, campaign pages, and support content. This included checking spacing, breakpoints, content length, and edge cases that surfaced when real CMS content was applied.

5 Handoff

Legacy-to-new migration specs for developers

The handoff focused on migration clarity: before-and-after references, style differences, responsive notes, and implementation details that helped developers update components incrementally without disrupting live pages.
